## 📌 Project Overview
This project presents a detailed audit and analysis of Git, a distributed version control system widely used in modern software development. The purpose of this work is to examine Git from both technical and philosophical perspectives, including its origin, licensing model, and its significance within the open-source ecosystem.

In addition to theoretical discussion, this project also includes hands-on work performed in a Linux environment along with shell scripts that demonstrate practical usage of open-source tools and command-line automation.

---

## 💡 Why I Chose Git
While working on programming assignments and small projects, managing multiple versions of code manually became confusing and inefficient. Tracking changes, reverting mistakes, and collaborating with others was difficult without a structured system. After learning Git, I found that it simplified version tracking and improved workflow organization.

Because Git is widely used and deeply connected to open-source development, I chose it for this audit. Studying Git in detail allowed me to understand both its technical functionality and the collaborative philosophy behind open-source tools.

## 🧾 Shell Scripts Included
This project includes five shell scripts demonstrating basic Linux automation and open-source tool usage:

### 🔹 Script 1: System Identity Report
Displays system information such as kernel version, logged-in user, uptime, and Linux distribution details.

### 🔹 Script 2: FOSS Package Inspector
Checks whether Git is installed on the system and prints the installed version.

### 🔹 Script 3: Disk and Permission Auditor
Analyzes selected directories and displays file permissions, ownership, and disk usage.

### 🔹 Script 4: Log File Analyzer
Reads a specified log file and counts occurrences of a user-provided keyword.

### 🔹 Script 5: Open Source Manifesto Generator
Accepts user input and generates a text file describing open-source principles.
